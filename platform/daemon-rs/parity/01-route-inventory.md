## Inventory method and caveats

- Route extraction looked for Hono `app.get/post/put/delete/patch(...)` registrations in the requested TypeScript files and matched them to Axum `.route(...)` registrations in `platform/daemon-rs/crates/signet-daemon/src/main.rs`.
- Rust path parameters use `{id}` while Hono paths use `:id`; matching treats parameter names as equivalent when the literal route shape matches.
- `native` means mounted plus `native-rust-replay-proven` in `route-parity.json`; `stub` and `partial` override that where Rust source shows placeholder/no-op or incomplete semantics.
- No route was classified `divergent`; clear non-matching/incomplete behavior was classified as `stub` or `partial` instead of guessing native parity.
