## TL;DR

The user's instinct is confirmed and quantified: **the Rust daemon is ~97% there
on API shape but materially behind on internal behavior.** Concretely —

- **API surface:** 318/318 TS routes have a Rust counterpart. 0 missing.
  route-parity.json (auto-generated by `bun scripts/check-rust-daemon-parity.ts --write`)
  marks **310 `native-rust-replay-proven` + 9 `mounted-shallow`** (all 9 are auth).
- **Behavioral gap:** **28 stubs + 7 partials** across routes; the subtle
  *behavioral cores* are where real divergence lives (see below).
- **Test coverage:** 167 TS test files → only **42 have a Rust equivalent;
  114 need porting** (11 are JS-runtime-only, out of scope).
- **Verification gap:** `signet-shadow` only diffs HTTP status + JSON body.
  Its rule fields (`tolerance`, `arrayOrdering`, `compareMode`, …) are
  **dead code** — never enforced (`compare.rs:30/42/55`). No internal-state,
  header, or streaming comparison exists.

## Behavioral cores — parity risk ranking (from `02`)

Mirror these most carefully. TS code IS the spec here.

| # | Core | Status | Why it's a risk |
|---|---|---|---|
| 1 | **Retention decay** | divergent | Rust uses wrong columns (`deleted=1` vs `is_deleted`/`deleted_at`) and wrong table (`memory_embeddings` vs `embeddings`); reduced cold snapshots. Looks like real bugs, not perf choices. |
| 2 | **Recall / ranking & scoring gates** | divergent | Rust has fast BM25/vec/boost/rerank, but missing TS gates: temporal candidates, traversal-primary fusion, SEC-lite, dampening, currentness, source fallbacks, summary budget, **authorize-candidates-before-content** boundary. Changes ordering + leakage. |
| 3 | **Pipeline stages** | partial | Rust worker only leases *extraction* jobs; missing controlled fact writes, escalation, contradiction handling, full stage orchestration. Worker bumps `facts_written` **without inserting fact memories** (`worker.rs:596`). |
| 4 | **agent_id / visibility scoping** | partial | Core filters exist but must be mirrored route-by-route. Pervasive security invariant. |
| 5 | **Memory writes & lineage / attribution** | partial | Route-level write primitives are native; pipeline attribution + autonomous writes not mirrored. |
| 6 | **Auth & secrets** | partial | Primitives native (tokens/roles/scope/rate-limit/mutation-frozen/encrypted store), but 9 shallow auth routes + secrets mounted directly (not capability-guarded like TS). |
| 7 | **Hook / hidden-message injection** | partial | Runtime-path 409 conflict native; session-start inject is a tiny subset of TS full inject (memory/identity/plugin/secret/recovery/budgeted recall). |
| 8 | **Ontology apply semantics** | native | ✅ `group_key`+`claim_key`, version roots, `entity_dependencies`, proposal evidence all native. Acceptable structural divergence. |

**Stub-markers:** 9 mounted-shallow auth routes (route-parity.json:135–220) +
score-tolerance/hook-content/secrets-names-only tolerances in parity-rules.json.

## Replay harness state (from `04`)

- `route-parity.json` + `parity-rules.json` + `schema.sql` exist.
- `contract_replay.rs` spins up a real daemon on an ephemeral port, seeds
  SQLite directly, replays. **Ignored by default** (needs built binary).
- `signet-shadow` = live proxy, forwards to both daemons, diffs status + JSON body.
- **Critical finding:** `parity-rules.json` fields (`tolerance`, `arrayOrdering`,
  `timestampPrecision`, `compareMode`, `compareBody`) are **deserialized but
  never enforced** — dead code in `compare.rs`. Must be wired before rules mean anything.
- No header / streaming / **internal-state** comparison anywhere.
