# 02 — Behavioral Cores Parity Inventory

Scope: subtle TypeScript daemon behaviors that are hard to re-derive in Rust. Status values follow the requested vocabulary: `native`, `stub`, `partial`, `missing`, `divergent`.

## Highest-risk cores first

1. **Retention decay — `divergent` / parity risk.** Rust retention targets legacy-looking columns/tables (`deleted`, `memory_embeddings`) while TS uses `is_deleted`, `embeddings`, cold snapshots, graph-link decrement, and vec cleanup (`platform/daemon/src/pipeline/retention-worker.ts:1`, `platform/daemon/src/pipeline/retention-worker.ts:131`, `platform/daemon/src/pipeline/retention-worker.ts:245`, `platform/daemon-rs/crates/signet-pipeline/src/retention.rs:191`, `platform/daemon-rs/crates/signet-pipeline/src/retention.rs:224`, `platform/daemon-rs/crates/signet-pipeline/src/retention.rs:242`).
2. **Pipeline stages — `partial` / parity risk.** Rust has native extraction/decision/retention/document/maintenance/summary modules, but the main worker only leases extraction jobs and does not mirror TS controlled fact writes, escalation, contradiction handling, or full stage orchestration (`platform/daemon/src/pipeline/worker.ts:1019`, `platform/daemon/src/pipeline/worker.ts:1156`, `platform/daemon/src/pipeline/worker.ts:1186`, `platform/daemon/src/pipeline/worker.ts:1290`, `platform/daemon-rs/crates/signet-pipeline/src/worker.rs:393`, `platform/daemon-rs/crates/signet-pipeline/src/worker.rs:543`, `platform/daemon-rs/crates/signet-pipeline/src/worker.rs:574`).
3. **Recall/search ranking and scoring gates — `divergent` / parity risk.** Rust implements fast BM25/vector/boost/rerank primitives, but TS recall has a broader staged scoring pipeline with hints, temporal candidates, traversal-primary fusion, authorization-before-content, SEC-lite, dampening, currentness, source fallbacks, and summary budget (`platform/daemon/src/memory-search.ts:997`, `platform/daemon/src/memory-search.ts:1162`, `platform/daemon/src/memory-search.ts:1203`, `platform/daemon/src/memory-search.ts:1378`, `platform/daemon/src/memory-search.ts:1634`, `platform/daemon/src/memory-search.ts:1679`, `platform/daemon/src/memory-search.ts:1754`, `platform/daemon/src/memory-search.ts:1834`, `platform/daemon-rs/crates/signet-core/src/search.rs:1`, `platform/daemon-rs/crates/signet-daemon/src/reranker.rs:1`).
4. **agent_id / visibility scoping — `partial` / parity risk.** Core filters and write guards exist in Rust, but this is pervasive and must be mirrored route-by-route; TS explicitly preserves an authorize-candidates boundary before content-bearing stages (`platform/daemon/src/memory-access-scope.ts:1`, `platform/daemon/src/memory-search.ts:999`, `platform/daemon-rs/crates/signet-core/src/search.rs:112`, `platform/daemon-rs/crates/signet-daemon/src/routes/write.rs:1130`, `platform/daemon-rs/crates/signet-daemon/src/auth/middleware.rs:248`).
5. **Auth & secrets — `partial` / parity risk.** Rust has token roles, permission checks, scope checks, rate limiting, mutation-frozen guards, and encrypted secret storage, but all auth UI/API-key/SSO/SAML routes are mounted-shallow and secrets routes are mounted directly rather than capability-plugin guarded like TS (`platform/daemon/src/routes/secrets-routes.ts:366`, `platform/daemon/src/routes/secrets-routes.ts:449`, `platform/daemon-rs/contracts/route-parity.json:6`, `platform/daemon-rs/contracts/route-parity.json:12`, `platform/daemon-rs/crates/signet-daemon/src/auth/types.rs:35`, `platform/daemon-rs/crates/signet-daemon/src/auth/middleware.rs:139`, `platform/daemon-rs/crates/signet-daemon/src/auth/middleware.rs:194`, `platform/daemon-rs/crates/signet-daemon/src/routes/write.rs:38`, `platform/daemon-rs/crates/signet-daemon/src/main.rs:910`).
6. **Hook / hidden-message injection — `partial` / parity risk.** Runtime path mutex and 409 conflict are native, but Rust session-start inject is much smaller than TS full memory/identity/plugin/secret/recovery inject; prompt-submit has Rust entity-context machinery but is not the same full TS path (`platform/daemon/src/routes/hooks-routes.ts:94`, `platform/daemon/src/routes/hooks-routes.ts:102`, `platform/daemon/src/hooks.ts:959`, `platform/daemon/src/hooks.ts:1100`, `platform/daemon-rs/crates/signet-daemon/src/routes/hooks.rs:39`, `platform/daemon-rs/crates/signet-daemon/src/routes/hooks.rs:47`, `platform/daemon-rs/crates/signet-daemon/src/routes/hooks.rs:1203`, `platform/daemon-rs/crates/signet-services/src/session.rs:124`).
7. **Memory writes & lineage / attribution — `partial` / parity risk.** Rust has native transaction wrappers, scoped dedupe, history, and artifact lineage, but pipeline fact writes are not mirrored in the Rust worker; route-level remember/write parity is stronger than worker-write parity (`platform/daemon/src/transactions.ts:1`, `platform/daemon/src/pipeline/worker.ts:378`, `platform/daemon/src/pipeline/worker.ts:514`, `platform/daemon/src/memory-lineage.ts:458`, `platform/daemon-rs/crates/signet-services/src/transactions.rs:45`, `platform/daemon-rs/crates/signet-services/src/transactions.rs:97`, `platform/daemon-rs/crates/signet-pipeline/src/memory_lineage.rs:727`, `platform/daemon-rs/crates/signet-pipeline/src/worker.rs:596`).
8. **Ontology apply semantics — `native` / acceptable structural divergence.** Rust implements proposal lifecycle plus native apply operations for entity/aspect/claim/link/merge and preserves `group_key` + `claim_key`, version roots, `entity_dependencies`, and proposal evidence; the monolithic Rust route file is structurally different but not a perf-only shortcut and is acceptable if kept semantically mirrored (`platform/daemon/src/ontology-proposals.ts:619`, `platform/daemon/src/ontology-proposals.ts:692`, `platform/daemon-rs/crates/signet-daemon/src/routes/ontology.rs:3218`, `platform/daemon-rs/crates/signet-daemon/src/routes/ontology.rs:3229`, `platform/daemon-rs/crates/signet-daemon/src/routes/ontology.rs:3485`, `platform/daemon-rs/crates/signet-daemon/src/routes/ontology.rs:3610`, `platform/daemon-rs/crates/signet-daemon/src/routes/ontology.rs:4024`, `platform/daemon-rs/crates/signet-daemon/src/routes/ontology.rs:4166`).
